## 进展 003：Exact Prefix Replay

日期：2026-07-15  
状态：已完成

### 对应提交

- `e1401d6`：`feat: add deterministic exact state replay`

### 完成内容

1. 定义显式 SKIP 与 assignment 的稳定 ActionKey。
2. 使用 scenario-relative 整数微秒编码时间，避免浮点时间身份漂移。
3. 定义 canonical task order、candidate order、prefix hash、schedule hash、candidate membership/order hash 与 state hash。
4. 从原 scenario 和 action prefix 重新枚举并恢复 Assignment，不从有损 schedule JSON 手工拼装状态。
5. trace provenance 记录 Git commit、关键实现文件 SHA-256 与 dirty 状态；代码来源不混入跨 solver 去重所用的物理状态身份。
6. smoke 子进程从输出目录启动，验证回放不依赖仓库工作目录。

### 当前验收证据

- 单元测试：29/29 通过。
- 真实 Sats1/M100 场景：10/10 个分层状态在两个全新 Python 进程中完全一致。
- 比较项包括 prefix、schedule、ordered candidate set、candidate membership、physical state、objective 与 state hash。
- 最大抽查候选集合为 1,538 个；恢复过程未修改源 schedule。
- 低 candidate cap 下打乱观测/通信窗口输入顺序，规范候选前缀保持一致。
- SKIP 前后 schedule hash 保持不变，但包含决策游标的 physical/state hash 正确变化。
- 新进程从输出目录而非仓库根目录启动，回放仍完全一致。

### 阶段边界

本阶段只提交 exact replay 相关文件与本日志，不混入用户原先暂存的 `方案.md`。真正的 CCOD 算法将在独立目录 `algorithms/ccod/` 中实现。
