# CCOD / EOS-Bench 论文方案评审

评审日期：2026-07-16（Asia/Shanghai）

评审对象：[`方案_v2.md`](./方案_v2.md) 的当前 CCOD 方案
实验状态：正式 100-state 包仍为零标签，不预报效果

## 总体结论

**Conditional Go。** 当前方案已经从旧 BCRD 的“通用双头 regret 蒸馏”收缩为更可辩护的 CCOD：以 EOS 冲突/资源 footprint 分配有限反事实查询，并用查询成本、容量平衡双尾覆盖和完整调度收益共同证伪。这个方向适合航天调度、智能优化或应用 AI venue；在完成预注册 gate 前，不足以支撑通用 ML 方法论文。

新颖性等级仍是 **Level 2 — High Overlap**。SeaRNN 已覆盖 forced alternatives、完成式 rollout、cost vector、uniform/policy/top-k 子采样和无 rollout 推理；LOLS/AggreVaTe 覆盖局部 cost-to-go；Herrmann–Schaub 已在 EOS 中把 MCTS action values 蒸馏为快速无搜索网络。当前唯一值得继续验证的方法性增量是：

> EOS constraint-consistent conflict/resource footprint 是否能在相同查询上限、相同 continuation 和相同学习器下，比 SeaRNN-style 与通用多样性 selectors 更充分地覆盖机会代价两端，并带来 held-out instances 的完整可行排程收益。

## 1. 当前方案的主要进步

1. **数据语义更准确**：不再把 SA/GA/ACO 最终解称为忠实 solver trajectories，而称为 `solution-induced constructive traces`。
2. **贡献边界更诚实**：明确不声称首次 rollout、cost-to-go、regret 或无搜索蒸馏，并把 SeaRNN、LOLS、AggreVaTe 与 Herrmann–Schaub 前置为威胁。
3. **方法从双头改为单头**：避免 CE logit 与另一个 regret head 的任意尺度拼接，直接学习已查询动作对的相对 order。
4. **selector 成为负重构件**：不再把 generic uncertainty/diversity 当创新，而要求 EOS footprint 相对 Uniform、Policy-sampling、Policy-top-K 和 generic farthest-first 给出同预算增益。
5. **指标修正为容量平衡双尾覆盖**：同时检查是否查询到好动作和坏动作，且只在有穷举 ground truth 的 dev states 计算。
6. **train→dev 边界清楚**：500-state 拟合状态只来自 train instances；dev 只做 selector、模型选择与闭环验证；sealed test 在方法锁定前不读取标签。
7. **资源路线现实**：Mac 完成 correctness、标签和 CPU gate；只有本地证据通过后才租 3090。
8. **执行状态语义严谨**：`complete`、`incomplete`、`invalid` 不再混成方法 pass/fail，避免把工程失败误写为科学 No-Go。

## 2. 摘要与主要创新审查

当前摘要的正确结构应保持为：问题缺口 → solution-induced traces → 固定上限 $B$ 的 EOS footprint query allocation → pairwise 单头蒸馏 → instance-level 公平评测 → 尚无正式结果。实验前版本不能写“显著提升”“优于基线”或任何预估数值。

建议最终保留三条贡献：

1. **可验证数据协议**：把异构 EOS 最终可行解转为带 semantic identity、prefix、candidate-set hash、schedule hash 和 objective identity 的精确构造状态。
2. **EOS-specific 查询分配**：用未来窗口删除、优先级质量损失、最后机会、姿态转移余量和资源瓶颈形成 conflict/resource footprint，在固定上限内选择 continuation queries。
3. **查询效率证据协议**：在相同 continuation 和模型下，联合报告 $\operatorname{BTailCov}_{16}$、pairwise ranking、完整 schedule、feasibility、online RT 与离线 query cost，并加入 footprint permutation 负对照。

以下只能列为实现选择或消融：单头 scorer、pairwise logistic loss、$H=5$、beam-8、集合编码器、内容寻址缓存和无搜索解码。

## 3. 方法公式与语义闭环

### 3.1 continuation value

强制动作应计入 $H$ 步视域，统一目标应与论文最终 schedule 指标完全相同：

$$
Q_H(s,a)=F_\omega\!\left(S_{t+H}^{a;\pi_L}\right)-F_\omega(S_t),
$$

其中 $\pi_L$ 是在任何标签产生前冻结的 deterministic continuation。不要额外混入程序性的 SKIP penalty；若 timeliness 原指标越小越好，则在 $F_\omega$ 中显式使用 $1-\mathrm{TM}$。

相对 observed action 的 advantage 为

$$
A_H(s,a)=Q_H(s,a)-Q_H\!\left(s,a^{\mathrm{obs}}\right).
$$

这只是稳定 target 定义，不是新颖性来源。

### 3.2 pairwise target

只保留差异超过预冻结容差的动作对：

$$
\mathcal P_s=
\left\{(a,b):a,b\in U_B(s),\ \left|A_H(s,a)-A_H(s,b)\right|>\epsilon_{\mathrm{pair}}\right\}.
$$

若 $|\mathcal P_s|=0$，该状态的 pairwise loss 定义为 0，并从“有监督状态”的平均分母排除；不能用除零、NaN 或任意 tie-break 产生伪标签。

### 3.3 EOS structural risk

footprint 的风险部分必须与真实未来 opportunity set 对齐，而不是使用模型不确定性换名。一个可复现定义是：

$$
r_s(a)=
\frac{
\sum_{m\in\mathcal M_L(s)}p_m
\dfrac{|D_L(s,a)\cap\mathcal O_m(s)|}{|\mathcal O_m(s)|}
}{
\sum_{m\in\mathcal M_L(s)}p_m
},
$$

其中只纳入 $|\mathcal O_m(s)|>0$ 的任务，$D_L(s,a)$ 是动作 $a$ 在未来 $L$ 内删除/压缩的合法机会集合；SKIP 的 structural risk 固定为 0。资源压力、transition slack 和 identity 等其余维度进入 diversity signature，不应重复计入同一风险标量。$L$ 必须在任何 $Q_H$ 前由无标签 profiler 冻结。

### 3.4 容量平衡双尾覆盖

令候选数为 $C_s$、selector 输出的预算集合为 $U_B(s)$，定义

$$
q_s=\max\!\left(1,\left\lceil0.1C_s\right\rceil\right),
\qquad
m_s=\min\!\left(q_s,\left\lfloor\frac{|U_B(s)|}{2}\right\rfloor\right).
$$

则

$$
\operatorname{BTailCov}_{B}(s)=\frac12\left[
\frac{\min\{|U_B(s)\cap\operatorname{Best}_{q_s}(s)|,m_s\}}{m_s}+
\frac{\min\{|U_B(s)\cap\operatorname{Worst}_{q_s}(s)|,m_s\}}{m_s}
\right].
$$

该指标依赖全动作 $Q_H$，只能在预注册 exhaustive dev states 上计算，不能用于非穷举 500-state 集。$B=16$ 时 selector 的首要任务是平衡覆盖两端，而不是只追逐 worst actions。

## 4. 新颖性边界

### 已有通用机制

- AggreVaTe：learner-state action exploration 与专家 cost-to-go；
- LOLS：一步偏离、终局 rollout loss 和弱 reference 下的 mixed rollout；
- SeaRNN：forced token、完整 cost vector、大词表动作子采样、测试时无 rollout；
- Deeply AggreVaTeD：可微神经 cost-to-go imitation；
- targeted exploration / active learning：非均匀查询的一般思想。

### 已有 EOS 机制

- Herrmann–Schaub：MCTS state-action values 蒸馏为快速 EOS 网络；
- Jacquet：EOS acquisition graph、GNN/PPO 与 PUCT MCTS；
- Mercado-Martínez：EOS conflict graph、GAT-DQN 候选 Q 与资源/图像质量目标；
- Belaid 2026：未知操作约束下主动查询 feasibility oracle；这是 EO query acquisition 的边界先例，但查询约束可行性而非备选动作 $H$ 步 value；
- EOS-Bench：统一场景、约束、求解器、结构难度和评测协议。

### 尚可守的部分

不是“把既有模块放到 EOS”这一事实，而是约束一致 footprint 是否带来可独立归因的 query-efficiency：

$$
\Delta_{\mathrm{selector}}
=
\operatorname{BTailCov}_{16}(\mathrm{CCOD})
-
\max_j\operatorname{BTailCov}_{16}(j),
$$

其中 $j$ 包括 Uniform、Policy-sampling、Policy-top-K 和 generic farthest-first。若该差异不足 +0.10、任一 dev instance 差值不为正，或 permutation 不回落，则撤回 selector 创新。

## 5. 实验协议审查

### 5.1 100-state 信号诊断

- 两个 dev instances 各 50 个唯一 actionable states；共 100 states / 1,570 frozen queries。
- 全部成功并通过 identity 才是 `complete`；未完成不能进入 signal 分母。
- signal-eligible 至少 80/100；Type-7 spread $\ge0.01$ 的总通过率至少 60%，每个 dev instance 至少 50%。
- 这一阶段只有两个独立 clusters，只作工程 Go/No-Go，不报告 CI、$p$ 值或统计显著性。

### 5.2 continuation 核验

- 至少 20 个 $17\le C_s\le128$ dev states 做 exhaustive $Q_H$。
- 固定 10 states 做 greedy–beam-8 排名比较；median Spearman 至少 0.70，undefined 即失败。
- beam-8 必须冻结 beam score、扩展顺序、稳定 tie-break、终止条件和 cap；不能在看结果后修改。

### 5.3 500-state train→dev selector gate

- 500 个拟合状态只从六个 train instances 抽取并按 `state_hash` 去重。
- 所有 selectors 共享 states、candidate sets、continuation、label cache、$B$、模型、loss、seeds 和训练步数。
- 主 selector 证据在 20 个 exhaustive dev states 上计算 mean $\operatorname{BTailCov}_{16}$；CCOD 相对最佳通用 selector 至少 +0.10，两个 dev instance 差值均严格为正。
- pairwise AUC 至少 0.55，Spearman 至少 0.10；任一 undefined 即失败。
- 两个 dev instance 的完整 schedule $F_\omega$ delta 均非负、均值严格大于 0，feasibility 100%。
- 只提高离线排序而不提高完整 schedule 趋势，判方法性 No-Go。

### 5.4 paper set 与最终统计

只有 500-state gate 通过后，才在新标签产生前冻结 3 个规模档、30–60 个独立 instances 的 semantic identity、stratified split、objective、source family、feature schema、continuation、$\epsilon_{\mathrm{pair}}$、seeds 和 evaluation manifest。

最终以 instance 为 cluster，比较 5 seeds；CCOD 相对最佳 sampled cost-vector baseline 的 paired $F_\omega$ difference 95% CI 必须大于 0，feasibility 不降，online RT 不超过 BC 的 1.2 倍。

## 6. 三态执行语义

| 状态 | 触发条件 | 允许的科学结论 |
|---|---|---|
| `complete` | 100 states、1,570 queries 全成功，identity 与冻结快照一致 | 才可计算 gate，并得到 `pass/go` 或 `fail/no_go` |
| `incomplete` | 未运行、query/state timeout、RSS、worker/process error、attempt exhausted、interrupted | `not_evaluated`，`method_decision=null`；可恢复执行 |
| `invalid` | identity mismatch、cache corrupt、hash/frozen/runner drift | `not_evaluated`，`method_decision=null`；粘性终态，换 run directory |

attempt budget 耗尽表示标签未完整，不表示方法失败。任何缺失 query 都不能静默跳过、替换状态或进入 gate 分母。

## 7. Mac 与 3090 的资源决策

当前不应租 3090。合理顺序是：

1. runner 源码稳定后重跑正式包的公开只读深验；
2. 只运行冻结前缀第一个 state，验收真实 cache、checkpoint、manifest、恢复和三态结果；
3. Mac 单 worker 完成 100/1,570；
4. 通过 signal 后完成 exhaustive 与 beam-8；
5. 实现所有通用 selectors，再实现 CCOD，完成 500-state train→dev gate；
6. 只有 64-state overfit、一个 seed CPU pilot、learned scheduler feasibility 和 dev closed-loop 全通过，才打包租 3090 训练。

3090 只加速已经证明值得扩大的训练，不负责发现方法是否有基本信号。

## 10. 论文结构建议

1. **引言**：selected-only supervision 的机会代价盲区；立即承认通用 L2S/SeaRNN 先例，将问题收窄到 EOS 查询分配。
2. **相关工作**：cost-to-go imitation、structured rollout subsampling、EOS neural scheduling/search、query-efficient selection 四条线。
3. **问题与数据协议**：solution-induced traces、semantic identity、可行动作 cap、统一目标和无泄漏 split。
4. **CCOD**：continuation、footprint、稳定 selector、pairwise scorer；明确每个构件哪些已有、哪些是当前组合。
5. **实验协议**：100/500/2,000 gates、exhaustive/beam validity、基线、成本核算和三态执行语义。
6. **结果**：先 selector-only，再 ranking，再完整 schedule，再 runtime/cost；阴性结果也按预注册规则报告。
7. **局限**：synthetic benchmark、solution-induced trace 偏差、bounded-horizon teacher、仅 30–60 instances、Mac 标签成本和外推限制。

## 11. 最终判断

- 航天调度 / 智能优化 / 应用 AI：**Conditional Go**。
- 通用 ML，按当前 Level 2 证据：**No-Go**。
- 若 footprint 相对最佳通用 selector 达到预注册双尾优势、负对照回落且最终完整 schedule CI 为正：可以重新评估更强 venue。
- 若 selector 优势成立但 closed-loop 不提升：转为 counterfactual labeling diagnostic/benchmark。
- 若 100-state opportunity-cost signal 不足：停止 CCOD，不租 GPU。
